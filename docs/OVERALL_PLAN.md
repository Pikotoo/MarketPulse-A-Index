# MarketPulse 整体方案

> 版本: v2.1 | 更新: 2026-07-17 | 状态: 核心功能可用 (70%)

---

## 一、产品定位

| 维度 | 答案 |
|------|------|
| **一句话** | A股投资者的每日数据终端——打开就知道今天市场怎么样 |
| **核心原则** | 帮你看清市场真实面目。不预测涨跌，只呈现数据背后的真相 |
| **差异化** | 一站式全维度 + 算法透明可审计 |
| **对标** | 韭圈儿恐贪指数（单维度）→ 我们做一站式 |
| **目标用户** | 个人投资者（日均看盘 5-10 分钟） |
| **使用场景** | 每天早盘前打开看一眼，盘中偶尔刷，收盘后复盘 |

---

## 二、合规边界

```
✅ 展示: 0-100 情绪分、分位数、比值、解读文字
❌ 禁止: 原始成交额、具体金额、个股名称、买卖建议
✅ 新闻: 只展示标题 + 来源链接，不爬正文
✅ AI: 只解释数据含义，绝不荐股
✅ 日历: 公开日程信息
✅ 相似期: 只列历史统计（"3次涨1次跌"），不暗示未来
❌ 龙虎榜: 已从方案移除（涉及个股）

数据原则: H:\数据大全\ 为唯一数据源，项目不存本地数据副本
```

## 三、信息架构（4 Tab）

```
Tab 1: 首页概览 (12 模块)
├── 综合情绪仪表盘（巨型，0-100，纯描述）
├── PE 温度计 + 历史分位
├── 北向资金情绪分
├── 融资融券情绪分
├── 两市量能分
├── 涨跌比情绪分
├── 限售解禁压力分（未来一周）
├── 板块资金流向
├── 宏观六维分解
├── 今日宏观日历
├── 今日市场要闻
└── 🕐 更新时间戳

Tab 2: 行业扫描
├── 行业热力图（32 行业 60 日动量着色）
├── 最强 Top5 / 最弱 Top5
├── 行业宽度变化趋势
├── 防御/进攻比
├── 行业拥挤度
└── 涨跌比趋势

Tab 3: 深度分析
├── 历史情绪曲线（可拖拽，7D/1M/3M/6M/1Y）
├── 历史相似期对比（纯统计陈列）
├── 指标对比器（双指标历史叠加）
├── 估值计算器（PE → 分位）
├── 风格轮动（大小盘/价值成长）
├── 跨资产联动（股债比/金价比/汇率）
├── 市场状态识别（七态）

Tab 4: 数据接入
├── API 文档
├── Key 管理
├── Python SDK 示例
├── 嵌入代码
└── 每日分享卡片生成
```

## 四、技术架构

### 前端

| 决策 | 选择 |
|------|------|
| 框架 | Vue 3 CDN（零构建，组件化） |
| 图表 | ECharts CDN |
| 状态管理 | Pinia CDN |
| 样式 | 暗色主题，手写 CSS |
| 适配 | 桌面 + 手机双端响应式 |
| AI 对话 | 底部固定栏，Flask 代理 DeepSeek + 数据上下文注入 |

### 后端

| 决策 | 选择 |
|------|------|
| 框架 | Flask |
| 数据存储 | SQLite（缓存 + Key 管理 + 审计） |
| 认证 | API Key + 分级限速 + 日配额 |
| 部署 | 本地 Windows，Flask 开发服务器（后续升级 WSGI） |

### 数据管线

```
akshare 下载 ──→ H:\数据大全\ (唯一写入位置)
                      │
                      ▼
             MarketPulse 信号引擎 (只读)
                      │
                      ▼
                 0-100 情绪分 → API 暴露
```

## 五、后端：21 个信号端点

### 5.1 估值维度（2 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 1 | `pe-percentile` | 沪深300 PE 在近5年历史中的分位（周频采样） | `csi300_pe.parquet` |
| 2 | `erp` | 1/PE - 10年国债收益率，估值温度补充 | PE + `CNG10Y.day` |

### 5.2 宏观维度（3 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 3 | `macro-score` | M2/PMI/CPI/PPI/SHIBOR/利差/汇率/M1M2/FDI 9维等权合成 | 9个 .day 文件 |
| 4 | `panic-index` | 行业波动率(0-40) + 板块离散度(0-30) + 宽度(0-30) | 32行业 .day |
| 5 | `liquidity-score` | SHIBOR水平 + 期限斜率 + M1M2增速差 + MLF-LPR利差 | SHIBOR/MLF 等 |

### 5.3 行业维度（5 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 6 | `sector-breadth` | 32行业站上MA60比例（0-100%），含7日趋势 | 32行业 .day |
| 7 | `defensive-ratio` | 防御板块 vs 进攻板块 60日动量差值 | 同上 |
| 8 | `sector-momentum` | 32行业60日涨跌幅完整排名 Top/Bottom | 同上 |
| 9 | `sector-heatmap` | 热力图数据：代码/名称/动量/MA60 状态/攻防标签 | 同上 |
| 10 | `sector-crowding` | Top5动量集中度 + 32行业离散度 + 趋势持续性 | 同上 |

### 5.4 宽度维度（2 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 11 | `advance-decline` | 上涨行业数/下跌行业数 + 市场情绪解读 | 32行业 .day |
| 12 | `new-high-low` | 创60日新高 vs 新低的行业数量比 | 同上 |

### 5.5 资金维度（5 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 13 | `margin-sentiment` | 融资YoY增速 + 融券比值 + 20日趋势，3维合成 | `RZ.day` `RQ.day` |
| 14 | `northbound-sentiment` | 月度净流向百分位 + 季度趋势 + 连续月份，3维合成 | akshare → `NORTH.day` |
| 15 | `volume-score` | 成交额偏离MA20 + 趋势变化 + 量价健康度 | `sh000300.day` |
| 16 | `lockup-pressure` | 解禁量占流通市值比的反向映射（解禁越多→压力分越高） | akshare → `LOCKUP.day` |
| 17 | `fund-sentiment` | 北向 + 融资 + ETF 三源综合 | 三个子端点合成 |

### 5.6 风格与跨资产（2 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 18 | `style-rotation` | 大小盘相对强弱 + 价值成长偏向 + 轮动速度 | 风格指数 .day |
| 19 | `cross-asset` | 股债比 + 金价比 + 人民币趋势综合 | 指数 + 宏观 .day |

### 5.7 状态与综合（2 个）

| # | 端点 | 算法概要 | 数据源 |
|---|------|----------|--------|
| 20 | `regime` | 恐慌/熊市/修复/中性/牛市/亢奋/极端 七态识别 | 多源合成 |
| 21 | `composite` | PE + ERP + 宏观 + 宽度 + 融资 + 北向 + 量能 7维等权 | 7个子端点合成 |
